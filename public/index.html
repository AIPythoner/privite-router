使用说明

  • 访问 /app1/xxx，会被转发到 路径前缀 = /app1 的目标服务器
  • 剥离前缀开启：/app1/foo → 目标/foo；关闭：保留原路径 目标/app1/foo
  • 最长前缀优先匹配，例如 /api/v2 比 /api 优先。
  • 全量转发：前缀填 / 即可把整域所有请求转到目标（适合后台系统，它们通常用 /static/... 绝对资源路径）。全量转发时"剥离前缀"无效。
  • 支持 HTTP / HTTPS / WebSocket。目标格式：http://ip:端口 或 https://host
  • 保留路径 /__admin、/healthz 不可作为前缀
  • 修改立即生效，无需重启服务
